# Architecture Diagrams

This folder contains comprehensive architecture diagrams for the MediGuard AI system. All diagrams are written in Mermaid syntax and can be viewed on GitHub, in Mermaid Live Editor, or any Mermaid-compatible viewer.

## Diagrams

1. **[System Architecture](01-system-architecture.md)** - High-level overview showing all layers (Frontend, API, Agents, Tools, Data, External Services)

2. **[Agent Workflow Sequence](02-agent-workflow.md)** - Sequential workflow showing how agents execute with tool calls and LLM interactions

3. **[Component Architecture](03-component-architecture.md)** - Detailed view of frontend and backend components and their relationships

4. **[Data Flow](04-data-flow.md)** - Shows how data flows from CSV files through DataFrames, tools, agents, and back to frontend

5. **[Complete Sequence Diagram](05-sequence-diagram.md)** - Detailed sequence diagram showing complete interaction flow from user input to results display

6. **[Technology Stack](06-technology-stack.md)** - Visual representation of the technology stack used in the project

## Viewing the Diagrams

### On GitHub
GitHub automatically renders Mermaid diagrams in markdown files. Simply view any `.md` file in this folder.

### Mermaid Live Editor
1. Copy the Mermaid code from any diagram file
2. Paste it into [Mermaid Live Editor](https://mermaid.live/)
3. The diagram will render automatically

### VS Code
Install the "Markdown Preview Mermaid Support" extension to view diagrams directly in VS Code.

## Usage

These diagrams can be used for:
- Documentation
- Presentations
- Architecture reviews
- Onboarding new developers
- System design discussions

## Updating Diagrams

When updating diagrams:
1. Edit the corresponding `.md` file
2. Test the diagram in Mermaid Live Editor
3. Ensure it renders correctly on GitHub
4. Update this README if adding new diagrams
